# 📋 Полная документация .env — TheBestMods

## 🔧 Общая информация
Файл `.env` содержит все конфигурационные параметры проекта.  
Для запуска скопируй `.env.example` в `.env` и заполни обязательные поля.

```bash
cp .env.example .env
```

---

## 🗄️ База данных

### `DATABASE_URL`
**PostgreSQL подключение.**  
```
postgresql://user:password@host:5432/database?schema=public&connection_limit=10
```
- `user` — пользователь БД (по умолч. `postgres`)
- `password` — пароль пользователя
- `host` — хост (в Docker: `postgres`, локально: `localhost`)
- `5432` — стандартный порт PostgreSQL
- `database` — имя БД (по умолч. `thebestmods`)
- `connection_limit=10` — макс. соединений в пуле Prisma

**Railway:** выдаётся автоматически в разделе Variables.  
**VPS:** создай БД через `CREATE DATABASE thebestmods;`

## ⚡ Redis

### `REDIS_URL`
**Redis подключение.**  
```
redis://localhost:6379
```
Используется для:
- Rate limiting (защита от брутфорса)
- Кеширования сессий
- Очереди задач (Bull)
- Временных кодов авторизации
- Socket.io адаптер

В Docker: `redis://redis:6379`

## 🔐 JWT (JSON Web Tokens)

### `JWT_SECRET`
**Секретный ключ для access-токенов.**  
- Минимум 64 символа (латиница, цифры, спецсимволы)
- Генерация: `openssl rand -hex 32`
- Пример: `a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4e5f6a1b2`

### `JWT_REFRESH_SECRET`
**Секретный ключ для refresh-токенов.**  
- Другой рандомный ключ (не такой же как JWT_SECRET!)
- Генерация: `openssl rand -hex 32`

### `JWT_ACCESS_EXPIRES_IN`
**Время жизни access-токена.**  
- Формат: `15m` (15 минут), `1h` (1 час)
- Рекомендация: 15 минут

### `JWT_REFRESH_EXPIRES_IN`
**Время жизни refresh-токена.**  
- Формат: `7d` (7 дней), `30d` (30 дней)
- Рекомендация: 7 дней

## 👑 Администраторы

### `ADMIN_SECRET_PATH`
**Секретный путь к админ-панели.**  
- НЕ ИСПОЛЬЗУЙ `admin` — это очевидно
- Пример: `x7k9m2p4q8w3`
- Итоговый URL: `https://site.com/x7k9m2p4q8w3`
- Путь проверяется через timing-safe сравнение (защита от перебора)

### `ADMIN_TELEGRAM_IDS`
**Telegram ID администраторов (через запятую).**  
- Пример: `123456789,987654321`
- Как узнать свой ID: напиши @userinfobot в Telegram
- Несколько ID — через запятую без пробелов

## 🔄 Сессии

### `SESSION_EXPIRY_HOURS`
**Время жизни сессии в часах.**  
- Рекомендация: `24` (1 день)
- Максимум одновременных сессий: 5 на пользователя

## 🚦 Rate Limiting

### `RATE_LIMIT_WINDOW_MS`
**Окно rate limiting в миллисекундах.**  
- `60000` = 60 секунд (1 минута)

### `RATE_LIMIT_MAX_REQUESTS`
**Макс. запросов за окно.**  
- `60` = 60 запросов в минуту для обычных эндпоинтов
- Система сама применяет разные лимиты:
  - `/api/auth/*` — 10 запросов/мин
  - `/api/admin/*` — 30 запросов/мин
  - Остальные — 60 запросов/мин

## 🌍 CORS

### `CORS_ORIGIN`
**Разрешённый источник (домен) для CORS.**  
- Локально: `http://localhost:3000`
- Прод: `https://thebestmods.com`
- Строгий режим — только один домен (не ставь `*`)

## 🔒 Шифрование

### `ENCRYPTION_KEY`
**AES-256 ключ шифрования (64 hex символа = 32 байта).**  
- Генерация: `openssl rand -hex 32`
- **ВАЖНО:** Если сменишь ключ — старые ключи не расшифруются!
- Храни backup этого ключа отдельно
- Используется для шифрования ключей читов в БД (encrypt-then-MAC)

## 📧 Email (SMTP)

### `SMTP_HOST`
**SMTP сервер.**  
- Примеры: `smtp.gmail.com`, `smtp.yandex.ru`, `smtp.mail.ru`

### `SMTP_PORT`
**Порт SMTP.**  
- `587` — TLS
- `465` — SSL
- `25` — без шифрования (не рекомендуется)

### `SMTP_USER`
**Пользователь SMTP.**  
- Email: `noreply@thebestmods.com`

### `SMTP_PASS`
**Пароль от SMTP или app-password.**  
- Для Gmail нужно создать пароль приложения

## 🐳 Docker

При запуске через `docker-compose up`:
- `DATABASE_URL` и `REDIS_URL` **НЕ НУЖНО** менять вручную
- docker-compose.yml сам подставляет правильные хосты:
  - `postgres://postgres:password@postgres:5432/thebestmods`
  - `redis://redis:6379`

Просто убедись что в `.env` нет закомментированных строк с этими переменными
(они переопределяются в environment секции docker-compose.yml)

## 🚀 Railway

При деплое на Railway:
1. Подключи **PostgreSQL** плагин → Railway сам выдаст `DATABASE_URL`
2. Подключи **Redis** плагин → Railway сам выдаст `REDIS_URL`
3. Все остальные переменные пропиши вручную в Variables
4. `NEXT_PUBLIC_SITE_URL` = URL твоего Railway приложения
5. Build command: `npm run build`
6. Start command: `npx prisma migrate deploy && npx prisma db seed && npm start`

## 🔐 Security Checklist

- [ ] `JWT_SECRET` и `JWT_REFRESH_SECRET` — разные, длинные (>64 символов)
- [ ] `ENCRYPTION_KEY` — сгенерирован через `openssl rand -hex 32`
- [ ] `ADMIN_SECRET_PATH` — сложный, не `admin`
- [ ] `ADMIN_TELEGRAM_IDS` — заполнен реальными ID
- [ ] `CORS_ORIGIN` — строгий, без `*`
- [ ] `RECAPTCHA_*` — настроена
- [ ] Никогда не коммитить `.env` в git (`.env` в `.gitignore`)
- [ ] Резервная копия `.env` в безопасном месте
